This class makes it easy to know which environment you're working on, useful for hybrid modules and SSR applications.

### Get "process.env" Property

```ts
import { Environment } from '@aracna/core'

// will log "MY_ENV_PROPERTY" value if defined, otherwise will return an empty string for safety
console.log(Environment.get('MY_ENV_PROPERTY'))
```

### Has "process.env" Property

```ts
import { Environment } from '@aracna/core'

// will log true if "MY_ENV_PROPERTY" is defined
console.log(Environment.has('MY_ENV_PROPERTY'))
```

### Import/Require

Useful for importing a module dynamically without including it into the bundle.

```ts
import { Environment } from '@aracna/core'

Environment.import('my_ecmascript_module')
Environment.require('my_commonjs_module')
```

### Getters

#### Positive

- isBlobDefined
- isDevelopment
- isDocumentDefined
- isFetchDefined
- isFileDefined
- isFormDataDefined
- isJest
- isProduction
- isTest
- isProcessDefined
- isWindowDefined

#### Negative

- isBlobNotDefined
- isNotDevelopment
- isDocumentNotDefined
- isFetchNotDefined
- isFileNotDefined
- isFormDataNotDefined
- isJestNotDefined
- isNotProduction
- isNotTest
- isProcessNotDefined
- isWindowNotDefined

#### Usage

```ts
import { Environment } from '@aracna/core'

if (Environment.isWindowDefined) {
  console.log('window is defined')
}
```

### NODE_ENV

```ts
import { Environment } from '@aracna/core'

// will log the "NODE_ENV" property value if defined
console.log(Environment.NODE_ENV)
```
